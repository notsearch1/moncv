Github est l'une des plateformes d'hébergement de code source les plus populaires. Est-ce là-bas permet aux développeurs de contribuer à des projets privés ou ouverts (mieux connu sous le nom de projets open source). 

Nous faisons programmeurs brésiliens Nous sommes venus aujourd'hui pour partager avec vous une liste de 10 référentiels que nous pensons être agréables pour votre programmeur.

La version Gihub pour Times est maintenant gratuite

Avant de commencer, nous, les programmeurs brésiliens, avons apporté une super nouvelle cette semaine. Github a annoncé mardi dernier (14) que sa version "pour les équipes" a été publiée gratuitement pour les utilisateurs.

Cela signifie que les équipes peuvent désormais gérer leur travail ensemble en un seul endroit : gestion de projet, révision de code, packages, etc. VÉRIFIEZ L'AFFAIRE ICI.

10 conseils sur le référentiel Github

REMARQUE : Cette liste a été extraite du blog "Meilleure programmation"hébergé sur la célèbre plate-forme "Medium".

1. Créez votre propre X

Étoiles GitHub : 61.300 XNUMX

cette référentiel fantastique il s'agit essentiellement d'une collection de tutoriels sur la façon de créer votre propre technologie. Il existe des exemples de création d'un outil de ligne de commande, d'un système d'exploitation, d'un moteur de recherche, d'un moteur de rendu 3D et bien d'autres.

Avez-vous déjà eu envie de créer votre propre langage de programmation ? Ou votre propre Docker ou Git ? Alors c'est le bon référentiel pour vous.

2. Livres de programmation gratuits

Étoiles GitHub : 139.000 XNUMX

Même si le nom de ce référentiel est Livres de programmation gratuits , il offre bien plus que cela. 

Il est disponible dans de nombreuses langues différentes et contient des sections pour des cours en ligne gratuits, des ressources de programmation interactives, des ensembles de problèmes et une programmation compétitive, ainsi que des podcasts et des terrains de jeux de programmation.

Mais la plupart traitent de livres de programmation. Et c'est vraiment une collection incroyable.

3. Oh mon zh

Étoiles GitHub : 106.000 XNUMX

le oh mon zsh est un framework open source et orienté communauté pour la gestion des configurations Zsh. Zsh est un shell interactif et un langage de script puissant que de nombreux développeurs utilisent.

Oh My Zsh est livré avec de puissants plugins et de beaux thèmes pour personnaliser votre configuration Zsh. 

Il faut un peu de travail pour le faire fonctionner, mais il existe d'excellents didacticiels sur le Web et des exemples d'autres développeurs qui vous aideront à trouver les bons paramètres pour vous.

4. Entretiens de l'Université de codage

Étoiles GitHub : 104.000 XNUMX

Ce référentiel est un plan d'études de plusieurs mois pour devenir ingénieur logiciel pour une grande entreprise comme Amazon, Google ou Facebook. 

Il s'adresse aux personnes qui débutent en génie logiciel (où des connaissances en informatique sont requises) et propose également des conseils sur la façon d'étudier pour devenir ingénieur en fiabilité ou ingénieur d'exploitation.

L'auteur a initialement créé ce référentiel en tant que liste de tâches personnelles pour accompagner votre processus d'apprentissage. 

Après avoir étudié huit à 12 heures par jour pendant plusieurs mois, il a finalement décroché l'emploi de ses rêves chez Amazon en tant qu'ingénieur en développement logiciel.

La Coding Interview University vous aidera à vous préparer à des entretiens techniques dans des entreprises comme Google, Microsoft, Facebook ou similaires. Faites-en de ce qui en à de meilleur.

5. Gitignore : une collection de modèles .gitignore

Étoiles GitHub : 97.100 XNUMX

Ce référentiel est exactement ce que son nom suggère : une collection de .gitignoremodèles utiles. Pour chaque nouveau projet que vous configurez en tant que référentiel GitHub, il devient obligatoire d'avoir un .gitignorefichier pour filtrer ce qui est téléchargé. 

Le contenu de ce fichier varie d'un projet à l'autre et d'une langue à l'autre. 

Le référentiel contient des modèles pour presque tous les langages ou frameworks, tels que Rails, Python, Perl, Laravel, Java et plus encore. Il existe même un modèle pour Fortran !

6 Conception du système primaire

Étoiles GitHub : 86,200 XNUMX

c'est un grand référentiel pour les ingénieurs logiciels qui vous aidera à apprendre à concevoir des systèmes à grande échelle. Cela vous aidera à devenir un meilleur ingénieur. Le référentiel fournit une collection organisée de ressources pour ce vaste sujet.

La conception du système étant souvent une composante nécessaire du processus d'entretien technique dans de nombreuses entreprises, ce référentiel peut également vous aider à préparer ces entretiens avec un guide d'étude, des conseils sur la façon d'aborder un entretien, des questions d'entretien avec des solutions, un apprentissage interactif et un codage interactif défis.

[+] Les pages GitHub utilisent désormais les actions par défaut. Vérifier les détails de l'intégration

7. API publiques

Étoiles GitHub : 73,100 XNUMX

API publiques est une excellente liste d'API gratuites que vous pouvez utiliser pour vos projets et applications. Il couvre divers sujets tels que les affaires, les dessins animés, les animaux, les actualités, la finance, les jeux et plus encore.

Il existe des API, comme les chats ou les chiens, qui fournissent des images de… chats ou chiens.

Mais il en existe aussi des utiles, comme l'API Gmail ou l'API Google Analytics.

C'est vraiment une vaste collection, alors vérifiez-la par vous-même.

8. L'art de la ligne de commande

Étoiles GitHub : 70.100 XNUMX

Savoir travailler avec la ligne de commande est souvent négligé par les développeurs, mais cela permet d'améliorer la productivité et la flexibilité en tant qu'ingénieur. 

Ce référentiel contient des notes et des conseils utiles sur l'utilisation de la ligne de commande lorsque vous travaillez sur Linux. 

Il existe des sections spécifiquement pour Windows ou macOS et, en général, les astuces s'appliquent à d'autres systèmes d'exploitation UNIX. Il s'adresse aussi bien aux débutants qu'aux plus expérimentés.

Bien que le référentiel ne semble plus être maintenu activement, il fournit toujours de très bons conseils qui peuvent vous aider à travailler avec la ligne de commande. Et vous pouvez toujours forker le référentiel et le conserver.

9. Algorithmes JavaScript et structures de données

Étoiles GitHub : 64,700 XNUMX

Ce référentiel contient des exemples de nombreux algorithmes et structures de données populaires pour JavaScript. Chaque exemple est noté débutant ou avancé pour indiquer la difficulté. 

Il existe des exemples de table de hachage, de pile, de file d'attente, de pile, de mathématiques, de chaînes, d'ensembles et plus encore.

10. Carte des développeurs

Étoiles GitHub : 98.600 XNUMX

O dernier référentiel contient un ensemble de graphiques illustrant différents chemins à suivre et technologies à adopter pour devenir ingénieur front-end, back-end ou dev-ops en 2020. Bien que cela semble un peu écrasant au début, c'est un guide utile pour ce qui est à la fois possible et nécessaire dans cette industrie en évolution rapide.

Le repo est mis à jour chaque année pour refléter les changements dans l'écosystème.

J'espère que vous trouverez ces référentiels aussi utiles que moi et que vous pourrez les utiliser pour devenir de meilleurs ingénieurs logiciels. 

Qu'avez-vous pensé de la liste ? Avez-vous un autre référentiel à indiquer ? Laissez-le dans les commentaires!
